# AI自由家 小程序前端迁移确认单

## 1. 目标
将 Flutter 版本 AI Freedom Home 在 unibest-main 中迁移为微信小程序前端版本，按 1:1 还原视觉与核心交互，最终可真实后端联调并可运行。

## 2. 你已确认的约束
1. 页面范围：全部页面都要完成。
2. 注释代码：Flutter 中已注释掉的功能暂不实现。
3. 首页媒体区：必须保留背景表现。
4. 首页媒体策略：先静态图占位。
5. 首页视频策略：异步获取视频地址，视频加载完成后按顺序循环播放。
6. 语言策略：首期仅中文。
7. 底部导航：Flutter 当前未启用的底部导航先不做。
8. 接口策略：对接真实后端接口，不使用 Mock。
9. 宣传图：作为宣传素材处理，不纳入业务页面主流程实现。
10. 推进顺序：同意按分阶段顺序推进。
11. 当前动作：先输出总结文档，待你确认后再开始开发。

## 3. 实施范围定义
### 3.1 必做页面模块（全量）
- 启动与登录模块
- AI自由家首页模块
- AI装修全流程模块
- 个人中心与设置模块
- 图库、日记、我的家模块
- 公共页面模块（关于、语言、国家码等）

### 3.2 暂不实现
- Flutter 源码中注释掉的功能点
- 与宣传展示相关但不属于业务流程的海报型页面

## 4. 分阶段执行顺序
### P0 基础搭建与运行骨架
- 路由重构与页面占位创建
- 全局样式变量、资源目录、主题基线
- 真实后端请求基础能力与鉴权链路
- 小程序端运行链路打通

### P1 核心业务闭环
- 登录与会话状态
- 首页静态背景 + 视频异步加载与循环播放
- AI装修任务主链路（类型、上传、风格/方案、生成中、结果、任务列表）
- 关键异常态与加载态

### P2 非核心但必做业务页面
- 个人中心、账号设置、偏好设置、反馈、关于
- 日记列表与新增
- 图库列表与详情
- 我的家页面

### P3 联调与验收收口
- 全链路真实接口联调
- 小程序体验优化（交互反馈、弱网处理、错误提示）
- 回归测试与问题修复

## 5. 首页 1:1 设计与交互落地方案
- 初始渲染：展示静态背景图（与 Flutter 资源一致）。
- 数据拉取：页面进入后异步请求首页视频素材接口。
- 资源准备：逐个预加载视频资源，未就绪前保持静态图。
- 播放切换：全部可播后进入顺序循环播放（空房设计 -> 现房改造 -> 软装换搭）。
- 文案联动：标题、副标题、去试试按钮行为与当前视频类型保持一致。

## 6. 验收标准
- 视觉：关键页面与 Flutter 版本对齐，布局、字号、色彩、切图位置无明显偏差。
- 交互：首页媒体区符合“静态占位 -> 异步加载 -> 顺序循环播放”策略。
- 功能：全部页面可访问，主流程可走通。
- 接口：真实后端可正常请求，核心接口成功率满足联调要求。
- 平台：微信小程序可运行。
